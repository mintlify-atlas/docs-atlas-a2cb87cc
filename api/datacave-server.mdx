# datacave-server

The `datacave-server` crate provides the main server binary, configuration management, authentication, sharding coordinator, and Raft-based replication.

## Module Structure

Located at `crates/datacave-server/src/`:

- **main.rs** - CLI entry point
- **config.rs** - Configuration types
- **server.rs** - Server implementation
- **coordinator.rs** - Sharding coordinator
- **auth.rs** - Authentication
- **failover.rs** - Failover logic
- **raft.rs** - Raft consensus

## CLI Commands

The server binary provides three commands.

**Source:** `crates/datacave-server/src/main.rs:15-36`

### serve

Start the database server.

```bash
datacave-server serve --config config.toml
```

**Options:**
- `--config <path>` - Path to configuration file (default: `config.example.toml`)

**Source:** `crates/datacave-server/src/main.rs:24-27`

### check-config

Validate configuration file.

```bash
datacave-server check-config --config config.toml
```

Parses and validates the configuration, then exits.

**Source:** `crates/datacave-server/src/main.rs:28-31`

### gen-password-hash

Generate Argon2 password hash for user configuration.

```bash
datacave-server gen-password-hash --password mypassword
```

Outputs an Argon2 hash suitable for the `password_hash` field in config.

**Source:** `crates/datacave-server/src/main.rs:32-36` and `main.rs:53-60`

**Example:**

```bash
$ datacave-server gen-password-hash --password secret123
$argon2id$v=19$m=19456,t=2,p=1$...
```

**Source:** `crates/datacave-server/src/coordinator.rs`

### route_plan

Routes a statement to appropriate shards.

**Source:** `crates/datacave-server/src/coordinator.rs:20-36`

**Routing Strategy:**
- **Read-only queries** (SELECT): Route to all shards
- **Write queries** (INSERT, UPDATE, DELETE, CREATE TABLE): Route to single shard based on table name hash

```rust
let coordinator = Coordinator::new(4);
let plans = coordinator.route_plan(&stmt);
for plan in plans {
    println!("Shard {}: {:?}", plan.shard_id, plan.stmt);
}
```

### aggregate

Combines results from multiple shards.

**Source:** `crates/datacave-server/src/coordinator.rs:38-66`

**Aggregation Strategy:**
- **Read queries**: Concatenates rows from all shards
- **Write queries**: Sums `rows_affected`

```rust
let results = vec![result1, result2, result3, result4];
let combined = coordinator.aggregate(&stmt, results);
println!("Total rows: {}", combined.rows.len());
```

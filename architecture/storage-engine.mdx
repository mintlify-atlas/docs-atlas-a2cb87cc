Datacave's storage engine is built on a Log-Structured Merge-Tree (LSM) architecture, optimized for write-heavy workloads with efficient compaction and optional encryption at rest.

## LSM Architecture Overview

The LSM engine consists of three main components:

```
┌─────────────┐
│   MemTable  │ ◄─── Writes (in-memory BTreeMap)
└──────┬──────┘
       │ Flush (when size threshold reached)
       ▼
┌─────────────┐
│  SSTable 1  │ ◄─── Immutable sorted files on disk
├─────────────┤
│  SSTable 2  │
├─────────────┤
│  SSTable 3  │
└─────────────┘
       │ Compaction (merge + dedup)
       ▼
┌─────────────┐
│ Compacted   │
│  SSTable    │
└─────────────┘

       ┌─────────────┐
       │     WAL     │ ◄─── Write-Ahead Log (durability)
       └─────────────┘
```

</ParamField>

## MemTable

The MemTable is an in-memory data structure using a `BTreeMap` for sorted key-value storage:

```rust filename="datacave-lsm/src/memtable.rs:3-24" showLineNumbers
#[derive(Debug, Default)]
pub struct MemTable {
    entries: BTreeMap<Vec<u8>, Vec<u8>>,
    bytes: usize,
}

impl MemTable {
    pub fn new() -> Self {
        Self {
            entries: BTreeMap::new(),
            bytes: 0,
        }
    }

    pub fn put(&mut self, key: Vec<u8>, value: Vec<u8>) {
        if let Some(prev) = self.entries.insert(key, value) {
            self.bytes = self.bytes.saturating_sub(prev.len());
        }
        if let Some((k, v)) = self.entries.last_key_value() {
            self.bytes = self.bytes.saturating_add(k.len() + v.len());
        }
    }
}
```

### Key Features

<AccordionGroup>
  <Accordion title="Sorted Storage" icon="arrow-down-a-z">
    BTreeMap maintains keys in sorted order, enabling efficient range queries and ordered iteration during flush.
  </Accordion>

  <Accordion title="Size Tracking" icon="gauge">
    Tracks approximate memory usage to trigger flush when `memtable_max_bytes` threshold is reached.
  </Accordion>

  <Accordion title="Fast Writes" icon="bolt">
    In-memory writes are O(log n) with no I/O overhead.
  </Accordion>
</AccordionGroup>

<Note>
The MemTable is protected by `Mutex<MemTable>` in the engine for thread-safe concurrent access.
</Note>

## Write-Ahead Log (WAL)

The WAL provides durability by logging all writes before applying them to the MemTable:

```rust filename="datacave-lsm/src/wal.rs:6-16" showLineNumbers
#[derive(Debug, Clone, Copy)]
pub enum WalOp {
    Put,
    Delete,
}

#[derive(Debug)]
pub struct Wal {
    file: File,
    encryptor: Option<DataEncryptor>,
}
```

### WAL Format

Each WAL record contains:

```
┌────────────┬─────────┬──────────┬─────┬─────────────┬───────┐
│ RecordLen  │  OpType │  KeyLen  │ Key │  ValueLen   │ Value │
│  (4 bytes) │ (1 byte)│ (4 bytes)│ ... │  (4 bytes)  │  ...  │
└────────────┴─────────┴──────────┴─────┴─────────────┴───────┘
```

### Append Operation

```rust filename="datacave-lsm/src/wal.rs:29-50" showLineNumbers
pub async fn append(&mut self, op: WalOp, key: &[u8], value: &[u8]) -> Result<()> {
    let op_byte = match op {
        WalOp::Put => 1u8,
        WalOp::Delete => 2u8,
    };
    let (key, value) = if let Some(encryptor) = &self.encryptor {
        (encryptor.encrypt(key)?, encryptor.encrypt(value)?)
    } else {
        (key.to_vec(), value.to_vec())
    };
    let key_len = key.len() as u32;
    let value_len = value.len() as u32;
    let record_len = 1 + 4 + key_len + 4 + value_len;
    self.file.write_u32_le(record_len).await?;
    self.file.write_u8(op_byte).await?;
    self.file.write_u32_le(key_len).await?;
    self.file.write_all(&key).await?;
    self.file.write_u32_le(value_len).await?;
    self.file.write_all(&value).await?;
    self.file.flush().await?;
    Ok(())
}
```

### Crash Recovery

On startup, the engine replays the WAL to rebuild the MemTable:

```rust filename="datacave-lsm/src/wal.rs:58-90" showLineNumbers
pub async fn replay(
    path: &str,
    encryptor: Option<DataEncryptor>,
) -> Result<Vec<(WalOp, Vec<u8>, Vec<u8>)>> {
    let mut file = OpenOptions::new().read(true).open(path).await?;
    let mut entries = Vec::new();
    loop {
        let len = match file.read_u32_le().await {
            Ok(v) => v,
            Err(_) => break,
        };
        if len == 0 {
            break;
        }
        let op = match file.read_u8().await? {
            1 => WalOp::Put,
            2 => WalOp::Delete,
            _ => WalOp::Put,
        };
        let key_len = file.read_u32_le().await? as usize;
        let mut key = vec![0u8; key_len];
        file.read_exact(&mut key).await?;
        let value_len = file.read_u32_le().await? as usize;
        let mut value = vec![0u8; value_len];
        file.read_exact(&mut value).await?;
        if let Some(enc) = &encryptor {
            key = enc.decrypt(&key)?;
            value = enc.decrypt(&value)?;
        }
        entries.push((op, key, value));
    }
    Ok(entries)
}
```

<Tip>
The WAL is reset (truncated) after each successful MemTable flush to SSTable, keeping the log file small.
</Tip>

## SSTable (Sorted String Table)

SSTables are immutable, sorted on-disk files containing key-value pairs:

```rust filename="datacave-lsm/src/sstable.rs:6-15" showLineNumbers
#[derive(Debug, Clone)]
pub struct SstEntry {
    pub key: Vec<u8>,
    pub value: Vec<u8>,
}

#[derive(Debug, Clone)]
pub struct SSTable {
    pub path: String,
}
```

### SSTable Format

Each entry is encoded as:

```
┌──────────┬─────┬────────────┬───────┐
│  KeyLen  │ Key │  ValueLen  │ Value │
│ (4 bytes)│ ... │  (4 bytes) │  ...  │
└──────────┴─────┴────────────┴───────┘
```

### Writing SSTables

```rust filename="datacave-lsm/src/sstable.rs:26-49" showLineNumbers
pub async fn write_with(
    path: &str,
    entries: &[SstEntry],
    encryptor: Option<&DataEncryptor>,
) -> Result<()> {
    let mut file = File::create(path).await?;
    let mut buffer = Vec::new();
    for entry in entries {
        let key_len = entry.key.len() as u32;
        let value_len = entry.value.len() as u32;
        buffer.extend_from_slice(&key_len.to_le_bytes());
        buffer.extend_from_slice(&entry.key);
        buffer.extend_from_slice(&value_len.to_le_bytes());
        buffer.extend_from_slice(&entry.value);
    }
    let bytes = if let Some(enc) = encryptor {
        enc.encrypt(&buffer)?
    } else {
        buffer
    };
    file.write_all(&bytes).await?;
    file.flush().await?;
    Ok(())
}
```

### Flush Operation

When MemTable reaches size threshold, it flushes to a new SSTable:

```rust filename="datacave-lsm/src/engine.rs:125-146" showLineNumbers
pub async fn flush(&self) -> Result<()> {
    let mut mem = self.memtable.lock().await;
    if mem.iter().next().is_none() {
        return Ok(());
    }
    metrics::counter!("lsm_flush_total").increment(1);
    let entries: Vec<SstEntry> = mem
        .iter()
        .map(|(key, value)| SstEntry {
            key: key.clone(),
            value: value.clone(),
        })
        .collect();
    let sst_path = format!("{}/sst-{}.db", self.options.data_dir, chrono_suffix());
    SSTable::write_with(&sst_path, &entries, self.encryptor.as_ref()).await?;
    self.sstables.lock().await.push(SSTable::new(sst_path));
    mem.clear();
    if self.options.wal_enabled {
        self.wal.lock().await.reset().await?;
    }
    Ok(())
}
```

<Info>
SSTable files are named with timestamps (e.g., `sst-1704067200000.db`) to maintain ordering and uniqueness.
</Info>

## Compaction

Compaction merges multiple SSTables into one, removing duplicates and deleted entries:

```rust filename="datacave-lsm/src/compaction.rs:6-23" showLineNumbers
pub async fn compact_tables(
    output_path: &str,
    tables: &[SSTable],
    encryptor: Option<&DataEncryptor>,
) -> Result<()> {
    let mut merged: BTreeMap<Vec<u8>, Vec<u8>> = BTreeMap::new();
    for table in tables {
        for entry in table.load_with(encryptor).await? {
            merged.insert(entry.key, entry.value);
        }
    }
    let entries: Vec<SstEntry> = merged
        .into_iter()
        .map(|(key, value)| SstEntry { key, value })
        .collect();
    SSTable::write_with(output_path, &entries, encryptor).await?;
    Ok(())
}
```

### Compaction Strategy

1. **Load all SSTables** into a BTreeMap (automatically deduplicates by key)
2. **Newest values win** (later SSTables overwrite earlier ones)
3. **Write compacted SSTable** with all merged entries
4. **Replace old SSTables** with single compacted file

```rust filename="datacave-lsm/src/engine.rs:148-159" showLineNumbers
pub async fn compact(&self) -> Result<()> {
    let tables = self.sstables.lock().await.clone();
    if tables.len() < 2 {
        return Ok(());
    }
    metrics::counter!("lsm_compact_total").increment(1);
    let output = format!("{}/sst-compacted-{}.db", self.options.data_dir, chrono_suffix());
    compact_tables(&output, &tables, self.encryptor.as_ref()).await?;
    info!("compacted {} tables into {}", tables.len(), output);
    *self.sstables.lock().await = vec![SSTable::new(output)];
    Ok(())
}
```

<Warning>
In production, compaction should run in the background to avoid blocking writes. Current implementation is triggered manually.
</Warning>

## Encryption at Rest

Datacave supports optional AES-256-GCM encryption for WAL and SSTable files:

```rust filename="datacave-lsm/src/encryption.rs:7-26" showLineNumbers
#[derive(Clone)]
pub struct DataEncryptor {
    cipher: Aes256Gcm,
}

impl DataEncryptor {
    pub fn new(key_bytes: &[u8]) -> Result<Self> {
        if key_bytes.len() != 32 {
            return Err(anyhow!("encryption key must be 32 bytes"));
        }
        let cipher = Aes256Gcm::new_from_slice(key_bytes)
            .map_err(|err| anyhow!("invalid key length: {err}"))?;
        Ok(Self { cipher })
    }

    pub fn encrypt(&self, plaintext: &[u8]) -> Result<Vec<u8>> {
        let mut nonce_bytes = [0u8; 12];
        rand::thread_rng().fill_bytes(&mut nonce_bytes);
        let nonce = Nonce::from_slice(&nonce_bytes);
```

### Encrypted Format

```
┌──────────┬────────────────┐
│  Nonce   │   Ciphertext   │
│ (12 bytes)│  (variable)   │
└──────────┴────────────────┘
```

- **Nonce**: Random 12-byte value for each encryption operation
- **Ciphertext**: AES-GCM encrypted data with authentication tag

<Tip>
Encryption is applied at the storage layer (WAL and SSTable), so all data at rest is protected when enabled.
</Tip>

## MVCC Integration

The LSM engine integrates with Datacave's MVCC system for snapshot isolation:

```rust filename="datacave-lsm/src/engine.rs:185-190" showLineNumbers
fn encode_versioned_key(key: &[u8], version: Version) -> Vec<u8> {
    let mut out = Vec::with_capacity(key.len() + 8);
    out.extend_from_slice(key);
    out.extend_from_slice(&version.to_be_bytes());
    out
}
```

Each key is suffixed with an 8-byte version number. Reads specify a snapshot version and retrieve the latest value less than or equal to that version:

```rust filename="datacave-lsm/src/engine.rs:192-202" showLineNumbers
fn is_key_match(versioned_key: &[u8], key: &[u8], snapshot: Version) -> bool {
    if versioned_key.len() < key.len() + 8 {
        return false;
    }
    let (prefix, version_bytes) = versioned_key.split_at(versioned_key.len() - 8);
    if prefix != key {
        return false;
    }
    let version = Version::from_be_bytes(version_bytes.try_into().unwrap_or([0u8; 8]));
    version <= snapshot
}
```

## Performance Characteristics

| Operation | Complexity | Notes |
|-----------|------------|-------|
| **Write (put)** | O(log n) + O(1) I/O | BTreeMap insert + WAL append |
| **Read (get)** | O(log n) + O(m) | Check MemTable, then scan m SSTables |
| **Flush** | O(n) | Serialize all MemTable entries to SSTable |
| **Compaction** | O(n log n) | Merge and sort all entries |

<CardGroup cols={2}>
  <Card title="Write Amplification" icon="chart-line">
    Low: writes go to WAL + MemTable, then SSTable on flush. Compaction adds additional writes.
  </Card>
  <Card title="Read Amplification" icon="magnifying-glass">
    Moderate: must check MemTable + all SSTables. Compaction reduces SSTable count.
  </Card>
</CardGroup>

### Extended Query Flow

The extended query protocol follows this sequence:

| Step | Message | Response |
|------|---------|----------|
| 1 | Parse (P) | ParseComplete (1) |
| 2 | Bind (B) | BindComplete (2) or ErrorResponse |
| 3 | Describe (D) Portal | NoData (n) or RowDescription |
| 4 | Execute (E) | RowDescription + DataRows + CommandComplete |
| 5 | Sync (S) | ReadyForQuery |

### Ready State Semantics

| State | Code | When |
|-------|------|------|
| Idle | `I` | Not in transaction; after COMMIT/ROLLBACK. |
| Transaction | `T` | After BEGIN; before COMMIT/ROLLBACK. |
| Error | `E` | After failed command; requires ROLLBACK. |

## PostgreSQL Compatibility Notes

### What Works

âś… **PostgreSQL clients**: `psql`, DBeaver, pgAdmin, language drivers

âś… **Wire protocol**: Simple Query and Extended Query flows

âś… **Connection lifecycle**: Startup, authentication, ReadyForQuery

### What Doesn't Work

âťŚ **PostgreSQL-specific syntax**: Many Postgres-specific features are not supported

âťŚ **System catalogs**: `pg_catalog`, `information_schema` not available

âťŚ **Advanced types**: Arrays, JSON, enums, ranges, etc.

âťŚ **Extensions**: No support for Postgres extensions

### Migration Considerations

When migrating from PostgreSQL to Datacave:

1. **Review schema**: Check that all column types are supported (INT, BIGINT, FLOAT, TEXT, BOOLEAN only)
2. **Remove constraints**: Foreign keys, CHECK, UNIQUE not supported (except PRIMARY KEY)
3. **Simplify queries**: No subqueries, window functions, CTEs
4. **Join types**: Only INNER and LEFT; rewrite RIGHT/FULL/CROSS joins
5. **Functions**: Remove function calls in WHERE/HAVING
6. **Test transactions**: Verify behavior without full ACID guarantees

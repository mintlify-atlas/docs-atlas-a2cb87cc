# datacave-sql

The `datacave-sql` crate provides SQL parsing, query planning, and execution with support for joins, aggregates, grouping, and ordering.

## Module Structure

Located at `crates/datacave-sql/src/`:

- **lib.rs:1-12** - Public exports
- **parser.rs** - SQL parsing
- **planner.rs** - Query plan generation
- **executor.rs** - Query execution
- **catalog.rs** - Catalog integration
- **vectorized.rs** - Vectorized execution

## SQL Parser

### parse_sql

Parses SQL text into AST statements.

**Source:** `crates/datacave-sql/src/parser.rs:6-10`


**Example:**


#### DELETE

**Source:** `crates/datacave-sql/src/executor.rs:383-427`

## Multi-Tenancy

The executor supports multi-tenancy through optional `tenant_id`:

- Keys are prefixed with tenant ID
- Row sequences are isolated per tenant
- `encode_row_key` creates tenant-scoped keys (executor.rs:444)

## Example Usage

```rust
use datacave_sql::{parse_sql, SqlExecutor};
use datacave_core::{Catalog, MvccManager};
use datacave_lsm::LsmEngine;
use std::sync::{Arc, Mutex};

let catalog = Arc::new(Mutex::new(Catalog::new()));
let mvcc = Arc::new(MvccManager::new());
let storage = Arc::new(LsmEngine::open(options).await?);

let executor = SqlExecutor::new(catalog, mvcc, storage);

let stmts = parse_sql("SELECT name, COUNT(*) FROM users GROUP BY name")?;
for stmt in stmts {
    let result = executor.execute(&stmt, None).await?;
    println!("Columns: {:?}", result.columns);
    println!("Rows: {:?}", result.rows);
}
```

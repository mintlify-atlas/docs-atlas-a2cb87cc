# datacave-core

The `datacave-core` crate provides foundational types, error handling, catalog management, and MVCC (Multi-Version Concurrency Control) primitives used throughout Datacave.

## Module Structure

Located at `crates/datacave-core/src/`:

- **lib.rs:1-10** - Public exports and module declarations
- **error.rs** - Error types
- **catalog.rs** - Schema catalog management
- **types.rs** - Core data types
- **mvcc.rs** - MVCC version management

## Error Handling

### DatacaveError

Main error enum for all Datacave operations.

**Source:** `crates/datacave-core/src/error.rs:3-15`

```rust
pub enum DatacaveError {
    #[error("catalog error: {0}")]
    Catalog(String),
    #[error("storage error: {0}")]
    Storage(String),
    #[error("sql error: {0}")]
    Sql(String),
    #[error("protocol error: {0}")]
    Protocol(String),
    #[error("not supported: {0}")]
    NotSupported(String),
}
```

**Variants:**
- `Catalog(String)` - Schema/catalog errors
- `Storage(String)` - LSM storage errors
- `Sql(String)` - SQL parsing/execution errors
- `Protocol(String)` - PostgreSQL protocol errors
- `NotSupported(String)` - Unsupported features

### Catalog

In-memory catalog for managing table schemas.

**Source:** `crates/datacave-core/src/catalog.rs:13-43`

```rust
pub struct Catalog {
    tables: HashMap<String, TableSchema>,
}

impl Catalog {
    pub fn new() -> Self
    
    pub fn create_table(&mut self, schema: TableSchema) 
        -> Result<(), DatacaveError>
    
    pub fn get_table(&self, name: &str) 
        -> Option<&TableSchema>
    
    pub fn list_tables(&self) -> Vec<TableSchema>
}
```

**Example:**

```rust
use datacave_core::{Catalog, TableSchema, Column};

let mut catalog = Catalog::new();
let schema = TableSchema {
    name: "users".to_string(),
    columns: vec![
        Column { name: "id".to_string(), data_type: "BIGINT".to_string() },
        Column { name: "name".to_string(), data_type: "TEXT".to_string() },
    ],
    primary_key: Some("id".to_string()),
};
catalog.create_table(schema)?;
```


### MvccManager

Manages version allocation and snapshot creation.

**Source:** `crates/datacave-core/src/mvcc.rs:10-31`

```rust
pub struct MvccManager {
    clock: AtomicU64,
}

impl MvccManager {
    pub fn new() -> Self
    
    pub fn next_version(&self) -> Version
    
    pub fn snapshot(&self) -> Snapshot
}
```

**Example:**

```rust
use datacave_core::mvcc::MvccManager;

let mvcc = MvccManager::new();
let v1 = mvcc.next_version();  // 1
let v2 = mvcc.next_version();  // 2
let snapshot = mvcc.snapshot(); // Captures current version
```

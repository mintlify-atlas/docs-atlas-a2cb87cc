## Welcome to Datacave

Datacave is a high-performance, distributed SQL database built in Rust from the ground up. It uses its own LSM-based storage engine and SQL layer, while speaking the PostgreSQL wire protocol for seamless client connectivity.

<Note>
  Datacave is designed for write-heavy workloads and modern distributed systems. Use existing PostgreSQL clients and tools while benefiting from custom storage and execution optimized for performance.
</Note>

## Architecture overview

Datacave is organized as a modular Rust workspace with clean crate boundaries:

- **datacave-core**: Core types, errors, catalog, and shared traits
- **datacave-lsm**: LSM storage engine with memtables, SSTables, and compaction
- **datacave-sql**: SQL parsing, query planning, and execution
- **datacave-protocol**: PostgreSQL wire protocol implementation
- **datacave-server**: Server binary and connection handling

Data flows from protocol handling → query planning → execution → LSM storage. This separation keeps components independently testable and replaceable.

## What's supported

Datacave supports a comprehensive subset of SQL features:

- **DDL**: CREATE TABLE with primary keys and common column types (INT, BIGINT, TEXT, BOOLEAN, FLOAT)
- **DML**: INSERT, SELECT, UPDATE, DELETE with WHERE clauses
- **Joins**: INNER JOIN and LEFT JOIN with ON or USING clauses
- **Aggregations**: COUNT, SUM, AVG, MIN, MAX with GROUP BY and HAVING
- **Query features**: ORDER BY, LIMIT, OFFSET
- **Transactions**: BEGIN, COMMIT, ROLLBACK (statement buffering until commit)

  Check the [SQL compatibility matrix](/sql/compatibility) for detailed feature status and limitations.

## Use cases

Datacave excels at:

- **High-throughput ingestion**: LSM architecture handles write-heavy workloads efficiently
- **Analytical queries**: Fast scans and aggregations over large datasets
- **Distributed systems**: Built-in sharding and replication for horizontal scaling
- **PostgreSQL migration**: Drop-in replacement for applications using standard PostgreSQL clients
